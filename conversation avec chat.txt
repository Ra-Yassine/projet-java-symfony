🥇 Stratégie 1 — Table unique + type (SINGLE TABLE)
👉 La plus simple et la plus utilisée en 1ère année

Principe
Une seule table USER

Un champ type_user

Les champs spécifiques peuvent être NULL

Schéma SQL
USER (
  id_user PK,
  nom,
  email,
  mot_de_passe,
  type_user ENUM('ETUDIANT','EMPLOYE','ADMIN'),
  niveau_etude,
  type_stage,
  experience,
  privileges
)
Avantages
✅ Simple à comprendre
✅ Simple à coder en Java (Servlet / JPA)
✅ Parfait pour projets académiques

Inconvénients
❌ Colonnes NULL
❌ Moins “pur UML”

📌 Recommandé pour ton projet

👉 La plus propre théoriquement (UML pur)

3️⃣ Exemples de jointures essentielles (niveau 1ère année)
🔹 1. Étudiant + Stage + Société
🎓 Afficher les stages demandés par un étudiant

SELECT u.nom, s.type_stage, so.nom AS societe
FROM stage s
JOIN user u ON s.id_etudiant = u.id_user
JOIN societe so ON s.id_societe = so.id_societe
WHERE u.type_user = 'ETUDIANT';
✔ Jointures claires
✔ Logique métier respectée

🔹 2. Employé + Offre + Entretien
💼 Afficher les entretiens d’un chercheur d’emploi

SELECT u.nom, o.titre, e.date_entretien
FROM entretien e
JOIN user u ON e.id_employe = u.id_user
JOIN offre o ON e.id_offre = o.id_offre
WHERE u.type_user = 'EMPLOYE';
🔹 3. Offres + Domaines
📂 Filtrer les offres par domaine

SELECT o.titre, d.nom_domaine
FROM offre o
JOIN domaine d ON o.id_domaine = d.id_domaine;
🔹 4. Paiement / Financement + Utilisateur

SELECT u.nom, f.montant, f.type_service
FROM financement f
JOIN user u ON f.id_user = u.id_user;
